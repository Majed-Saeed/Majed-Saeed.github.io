3:T9a7,<p>A public S3 bucket is one of those mistakes that is easy to make and expensive to leave. Instead of hoping a review catches it, you can wire AWS itself to notice and fix it.</p>
<h2>The shape of the solution</h2>
<p>Three pieces do the work:</p>
<ul>
<li><strong>AWS Config</strong> evaluates buckets against a rule and marks a public one as non-compliant.</li>
<li><strong>EventBridge</strong> routes that finding to compute.</li>
<li><strong>Lambda</strong> re-applies block-public-access and reports what it did.</li>
</ul>
<h2>The remediation function</h2>
<p>The function receives the resource id, applies the fix, and notifies. The core is small:</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">import</span> boto3

s3 = boto3.client(<span class="hljs-string">&quot;s3&quot;</span>)
sns = boto3.client(<span class="hljs-string">&quot;sns&quot;</span>)

<span class="hljs-keyword">def</span> <span class="hljs-title function_">handler</span>(<span class="hljs-params">event, context</span>):
    bucket = event[<span class="hljs-string">&quot;detail&quot;</span>][<span class="hljs-string">&quot;resourceId&quot;</span>]
    s3.put_public_access_block(
        Bucket=bucket,
        PublicAccessBlockConfiguration={
            <span class="hljs-string">&quot;BlockPublicAcls&quot;</span>: <span class="hljs-literal">True</span>,
            <span class="hljs-string">&quot;IgnorePublicAcls&quot;</span>: <span class="hljs-literal">True</span>,
            <span class="hljs-string">&quot;BlockPublicPolicy&quot;</span>: <span class="hljs-literal">True</span>,
            <span class="hljs-string">&quot;RestrictPublicBuckets&quot;</span>: <span class="hljs-literal">True</span>,
        },
    )
    sns.publish(
        TopicArn=TOPIC_ARN,
        Subject=<span class="hljs-string">&quot;S3 public access remediated&quot;</span>,
        Message=<span class="hljs-string">f&quot;Re-applied block-public-access to <span class="hljs-subst">{bucket}</span>&quot;</span>,
    )
</code></pre><h2>Why notify on every action</h2>
<p>The temptation is to let automation run silently. Don&#39;t. Publishing each change to SNS keeps the system auditable and gives a human the chance to review or roll back. Automated remediation earns trust by being legible, not by being invisible.</p>
<blockquote>
<p>Keep the rule set small and well understood. A guardrail you can reason about is worth more than a clever one you can&#39;t.</p>
</blockquote>
